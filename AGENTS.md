# ios-study 项目协作规范

## 项目定位
- 这是一个面向前端开发者的 iOS 成长路线站点。
- 当前正在从旧结构 `guide / objc / swift / xcode / practice` 重构到新结构：
  - `roadmap`
  - `starter`
  - `uikit`
  - `swift`
  - `objc-maintenance`
  - `advanced`

## 当前开发优先级
- 先做信息架构和学习路径重构，再补充缺失内容页。
- 第一优先级：
  - 首页与导航语义统一
  - 旧路径兼容
  - 正文内部跳转统一
- 第二优先级：
  - UIKit 主线内容补齐
  - Objective-C 老项目维护主线补齐
  - 高级进阶工程化主题补齐

## 内容设计原则
- 不按“语言知识点大全”扩写，按“阶段成长路径”扩写。
- 每个页面优先回答：
  - 这部分适合谁
  - 解决什么问题
  - 和前端对应关系是什么
  - 学完后下一步该去哪
- Objective-C 必须长期保留，不能被 Swift 路线挤掉。
- UIKit 必须作为独立主线存在，不能散落在 Xcode 和实战页面里。

## 代码结构约定
- 导航与侧边栏统一维护在 `src/data/config.ts`。
- 路由统一维护在 `src/App.tsx`。
- 首页阶段地图统一维护在 `src/pages/Home.tsx`。
- 新增内容页时，优先补齐：
  - 路由
  - 导航入口
  - 上一篇 / 下一篇
  - 所属阶段语义

## 文档同步规则
- Agent 需要自行判断本次改动是否已经影响项目对外说明或项目长期记忆，而不是机械地每次都改文档。
- 如有必要，Agent 应同步修改：
  - `README.md`
  - `PROJECT_MEMORY.md`
- 判断标准：
  - 如果改动影响项目定位、使用方式、目录结构、对外入口、开发命令、协作方式，应考虑修改 `README.md`。
  - 如果改动影响当前阶段、重构进度、兼容策略、后续计划、重要约定，应考虑修改 `PROJECT_MEMORY.md`。
  - 如果只是局部代码实现、小范围样式调整、未改变项目认知或协作上下文，可以不改这两个文件。

## 兼容策略
- 旧路径暂时保留兼容，不要直接删除旧路由，除非已经完成全站内部链接迁移。
- 如果新增新语义路径：
  - 优先加别名路由承接旧页面
  - 再逐步把正文和导航切到新路径

## 验证要求
- 每轮改动后至少执行：
  - `npm run lint`
  - `npm run build`
- 如果改的是信息架构，还要额外检查：
  - 顶部导航
  - 侧边栏归类
  - 首页入口
  - 上一篇 / 下一篇链接
